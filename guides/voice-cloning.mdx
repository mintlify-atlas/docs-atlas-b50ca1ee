---
title: "Voice Cloning"
description: "Clone any voice from a short reference audio clip using OmniVoice's zero-shot voice cloning."
---

Voice cloning mode reproduces the speaking style, timbre, and prosody of a reference speaker. You provide a short audio clip and its transcript — OmniVoice handles the rest. No fine-tuning or speaker enrollment is required.

<Note>
  Output audio is always 24 kHz mono. The returned value is a list of `torch.Tensor` objects with shape `(1, T)`.
</Note>

## Basic Example

Provide `ref_audio` (a path to a WAV file) and `ref_text` (its transcript):

```python
from omnivoice import OmniVoice
import torch
import torchaudio

model = OmniVoice.from_pretrained(
    "k2-fsa/OmniVoice",
    device_map="cuda:0",
    dtype=torch.float16,
)
# Apple Silicon: use device_map="mps" instead

audio = model.generate(
    text="Hello, this is a test of zero-shot voice cloning.",
    ref_audio="ref.wav",
    ref_text="Transcription of the reference audio.",
)

torchaudio.save("out.wav", audio[0], 24000)
```

`audio` is a list with one element per input text. Index into it with `audio[0]` when passing a single string.

## Auto-Transcription with Whisper

If you omit `ref_text`, OmniVoice automatically transcribes the reference clip using Whisper ASR. This is convenient when you do not have a transcript on hand:

```python
audio = model.generate(
    text="Hello, world.",
    ref_audio="ref.wav",
    # ref_text omitted — Whisper transcribes ref.wav automatically
)
```

<Tip>
  Auto-transcription adds a small overhead the first time it runs (Whisper model download + inference). If throughput matters, supply `ref_text` explicitly.
</Tip>

## Reusable VoiceClonePrompt

When generating many samples with the same reference voice, pre-compute a `VoiceClonePrompt` once and reuse it. This avoids re-tokenizing the reference audio on every call:

```python
# Compute once
prompt = model.create_voice_clone_prompt(
    ref_audio="ref.wav",
    ref_text="Transcription of the reference audio.",
)

# Reuse for many utterances
lines = [
    "First sentence to synthesise.",
    "Second sentence to synthesise.",
    "Third sentence to synthesise.",
]

for line in lines:
    audio = model.generate(text=line, voice_clone_prompt=prompt)
    # ... save audio
```

`voice_clone_prompt` overrides `ref_audio` and `ref_text` when all three are provided.

## Batch Voice Cloning

Pass a list of texts to generate multiple audio files in a single forward pass:

```python
texts = [
    "The quick brown fox jumps over the lazy dog.",
    "She sells seashells by the seashore.",
    "How much wood would a woodchuck chuck?",
]

audios = model.generate(
    text=texts,
    ref_audio="ref.wav",
    ref_text="Transcription of the reference audio.",
)

for i, audio in enumerate(audios):
    torchaudio.save(f"out_{i}.wav", audio, 24000)
```

<Info>
  Each item in `texts` is synthesised independently with the same reference voice. Batch size is only limited by GPU memory.
</Info>

## Reference Audio Quality Tips

<CardGroup cols={2}>
  <Card title="Keep it short" icon="scissors">
    Clips of 3–10 seconds work best. Very long recordings are automatically trimmed at the largest silence gap when `ref_text` is not provided. If you supply `ref_text` for a clip longer than 20 seconds, a warning is logged and trimming is skipped.
  </Card>
  <Card title="Silence is removed" icon="volume-xmark">
    Leading, trailing, and mid-sentence silences are automatically stripped before tokenisation. You do not need to pre-edit the file.
  </Card>
  <Card title="Clean audio" icon="waveform-lines">
    Minimize background noise, music, and reverb. A close-mic or headset recording produces noticeably better results than a room mic.
  </Card>
  <Card title="Consistent volume" icon="gauge">
    The model normalises quiet recordings automatically (RMS below 0.1 is rescaled), but heavily clipped or compressed audio can degrade quality.
  </Card>
</CardGroup>

## Non-Verbal Sounds

Insert expressive non-verbal tags directly in the synthesis text. They can appear at the start, end, or mid-sentence:

```python
audio = model.generate(
    text="[laughter] You really got me. I didn't see that coming at all.",
    ref_audio="ref.wav",
    ref_text="Transcription of the reference audio.",
)
```

### Supported Tags

| Tag | Description |
|-----|-------------|
| `[laughter]` | Natural laugh |
| `[confirmation-en]` | Confirming "uh-huh" |
| `[question-en]` | Rising questioning sound |
| `[question-ah]` | Questioning "ah?" |
| `[question-oh]` | Questioning "oh?" |
| `[question-ei]` | Questioning "ei?" |
| `[question-yi]` | Questioning "yi?" |
| `[surprise-ah]` | Surprised "ah!" |
| `[surprise-oh]` | Surprised "oh!" |
| `[surprise-wa]` | Surprised "wa!" |
| `[surprise-yo]` | Surprised "yo!" |
| `[dissatisfaction-hnn]` | Displeased "hnn" |
| `[sniff]` | Audible sniff |
| `[sigh]` | Audible sigh |

Tags can be combined with normal speech and with each other:

```python
audio = model.generate(
    text="[sigh] I knew this would happen. [laughter] Though I can't say I'm surprised.",
    ref_audio="ref.wav",
)
```

## Pronunciation Control

### Chinese — Pinyin with Tone Numbers

For Chinese polyphonic characters (characters whose pronunciation depends on context), insert the pinyin syllable with a tone number (1–4) immediately after the character or word:

```python
# 折 can be read as zhē (fold), zhé (discount/break), or zhě (person suffix)
# 折SHE2 → shé  (losing money sense)
# 折ZHE2 → zhé  (discount sense)
# 折ZHE1 → zhē  (fold sense)
audio = model.generate(
    text="这批货物打ZHE2出售后他严重SHE2本了，再也经不起ZHE1腾了。",
    ref_audio="ref.wav",
)
```

The pinyin annotation overrides the model's default grapheme-to-phoneme prediction for that character only. The rest of the text is processed normally.

### English — CMU Pronunciation Dictionary

For English words with irregular or ambiguous pronunciations, use CMU dictionary phoneme strings written in **uppercase** and wrapped in **square brackets**:

```python
# Override pronunciation of a single word mid-sentence
audio = model.generate(
    text="You could probably still make [IH1 T] look good.",
    ref_audio="ref.wav",
)
```

Phoneme symbols follow the [CMU Pronouncing Dictionary](https://svn.code.sf.net/p/cmusphinx/code/trunk/cmudict/cmudict.0.7a) notation (ARPAbet). Stress digits 0, 1, 2 are supported.

<Warning>
  Pronunciation overrides apply to the bracketed token only. Surrounding words are unaffected. Ensure the phoneme string is valid ARPAbet — invalid symbols may cause unexpected output.
</Warning>
